# TF2CFG

Personal TF2 config, restructured for readability.

## Install

Copy everything from this repo into `tf/cfg/` (yes, including the `cfg/`
subfolder — the class configs and `autoexec.cfg` must stay at the root of
`tf/cfg/` because the engine looks for them there by name).

## Structure

```
TF2CFG/
├── autoexec.cfg          # entry point — aliases + load order ONLY
├── scout.cfg ... spy.cfg # per-class binds, auto-loaded by the engine on class pick
├── config.cfg             ┐
├── config_default.cfg     │  engine-managed files, don't edit these directly,
├── motd_entries.txt       │  they're expected at cfg root by the game itself
├── server_blacklist.txt   │
├── server_custom.cfg      │
├── sixense_bindings.cfg   │
├── user.scr               ┘
└── cfg/                   # everything below is safe to reorganize
    ├── core/               # AutoSettings, MouseSettings, reset_binds
    ├── graphics/           # BadGraphics / ExcellentGraphics presets
    ├── viewmodel/          # VMOn / VMOff
    ├── hud/                # combat text on/off
    ├── crosshair/          # crosshair style presets
    ├── movement/           # mousewheel bhop on/off
    ├── sound/              # SoundButtons (volume/voice hotkeys)
    └── misc/               # cheater-spec, thirdperson guide, maxhp cheats
```

Rule of thumb: **only `autoexec.cfg` and the class files live at the root**.
Everything else lives in `cfg/<module>/` and is `exec`'d by full path
(`exec "cfg/graphics/BadGraphics"`), never exec'd from more than one place.

## What changed from the old layout

- **`autoexec.cfg` is now a pure orchestrator.** It used to mix unbinds,
  20+ aliases, module execs, cvar overwrites and binds in one 90-line file
  with no sections. Same content, but grouped and moved into `cfg/`.
- **`AutoSettings.cfg` no longer re-execs `VMOn`, `MouseSettings` and
  `SoundButtons`.** Those were already exec'd by `autoexec.cfg` right
  before `AutoSettings`, so they ran twice on every launch for no reason.
  `AutoSettings.cfg` is cvars only now.
- **Added `bind "RCTRL" "exec cfg/core/AutoSettings"`** as an explicit,
  version-controlled quick-reload bind. The old repo had this bind sitting
  only inside the engine-autosaved `config.cfg`, which is regenerated by
  the game on every exit — so it wasn't actually guaranteed to survive.
- Every file that got moved had its callers updated (`exec` paths in
  `autoexec.cfg`, the class configs, and `reset_binds.cfg`).

## Notes

- `ExcellentGraphics` is commented out in `autoexec.cfg` in favor of
  `BadGraphics` — swap the comment to flip presets.
- Crosshair aliases (`SetCrosshairDot`, etc.) still live in `autoexec.cfg`
  since they're one-liners that just wrap `exec` + a color alias; the
  actual `cl_crosshair_*` values live in `cfg/crosshair/`.
